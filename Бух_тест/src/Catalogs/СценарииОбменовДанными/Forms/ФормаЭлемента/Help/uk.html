Основне призначення сценарію - це виконання синхронізації даних автоматично за заданим розкладом. Сценарій необхідний для завдання розкладу і порядку надсилання і отримання даних.

Не рекомендується виконувати тільки відправку даних або лише отримання даних, хоча сценарій дозволяє це зробити. Рекомендується виконувати повноцінну синхронізацію даних в одному сценарії.

Настройка сценарія синхронізації

Введіть Найменування сценарію. Необхідно вказати найменування баз даних, що беруть участь в синхронізації, редакцію програми, для того щоб користувачі могли зрозуміти призначення даного сценарію.

Настройка сценарія синхронізації даних складається з наступних етапів:

  • Настроїти розклад виконання сценарію синхронізації даних.
    • Увімкніть прапорець Автоматично за розкладом. Після цього стає доступним для вибору гіперпосилання (По умовчанню дорівнює Розклад не задано). Натисніть гіперпосилання для того щоб задати розклад. Після підтвердження розкладу гіперпосилання показує складений розклад.
  • Вибрати інформаційні бази, для яких буде виконуватися сценарій синхронізації даних (як правило, сценарій налаштовується для обміну з однією інформаційною базою і включає відправку і отримання даних). Для кожної інформаційної бази вибрати спосіб підключення та виконувану дію (надсилання чи одержання). Для цього натисніть Додати, заповніть відповідні колонки:
    • Інформаційна база - Виберіть тип даних. Потім із запропонованого списку потрібного типу виберіть інформаційну базу;
    • З'єднання - виберіть спосіб підключення зі списку (залежить від властивостей вибраної інформаційної бази);
    • Виконувана дія - з допомогою кнопки Виберіть Завантаження або Вивантаження даних;
    • Напишіть Коментар - короткий опис сценарію.

Виконання сценарію

Натисніть Виконати сценарій. Буде зроблена спроба синхронізації даних. Перераховані в сценарії дії будуть виконуватися послідовно. Після того як синхронізація даних проведена, невдалі завантаженя або вивантаження даних позначаються значком в колонці Інформаційна база.
