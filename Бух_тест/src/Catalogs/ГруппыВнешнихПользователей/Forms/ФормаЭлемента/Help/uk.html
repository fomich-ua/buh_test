Група зовнішніх користувачів

Перегляд і редагування властивостей групи зовнішніх користувачів.

Зовнішніх користувачів можна об'єднувати в групи за різними ознаками. Група зовнішніх користувачів може включати однорідних користувачів, наприклад, контрагентів або фізичні особи. Також можна створити групу зовнішніх користувачів, яка включає різних зовнішніх користувачів зі схожими властивостями, наприклад, респондентів або учасників опитування. Так зручніше настроїти права доступу.

Введення групи зовнішніх користувачів

  • Введіть Найменування групи.
  • У полі Учасники групи для групи зовнішніх користувачів можна встановити тип учасників. Група може включати:
    • зовнішніх користувачів різних типів - для цього виберіть Будь-які користувачі;
    • зовнішніх користувачів певного типу (наприклад, контрагентів), для цього виберіть Будь-які користувачі заданого виду і один із списків для підбору учасників.
    • всіх зовнішніх користувачів певного типу (наприклад, всіх контрагентів) - для цього виберіть пункт Усі користувачі заданого виду і один із списків для підбору учасників. У цьому випадку поле Підібрати стає недоступним. Всі зовнішні користувачі з вказаного списку входять в таку групу за замовчуванням.
  • При редагуванні складу зовнішніх користувачів групи потрібно враховувати, що вони входять до складу груп-батьків.
  • Для додавання великої кількості зовнішніх користувачів можна використовувати команду Підібрати. У цьому випадку можна виділити кілька зовнішніх користувачів одночасно і включити їх в групу.
  • При використанні команди Додати можна включити в групу лише одного зовнішнього користувача.
  • Можна підпорадкувати нову групу одній з існуючих груп. Для цього в полі В групі виберіть потрібну групу зі списку.
  • В полі Коментар можна написати докладний опис даної групи.

Права доступу групи зовнішніх користувачів

В залежності від складу програми налаштовуються одним із способів:

  • Натисніть Права доступу на панелі навігації групи. Натисніть Включити до групи, виберіть групу доступу зі списку.
  • Якщо використовується безпосереднє призначення ролей, їх можна призначати не тільки для зовнішнього користувача, але і для групи зовнішніх користувачів.
