Опис властивостей тому зберігання файлів.

При створенні нового томи автоматично перевіряється доступ до зазначеного в полі Повний шлях до каталогу, у разі помилки виводиться відповідне повідомлення. Вже створеному тому не можна поміняти шлях.

Для кожного тома необхідно вказати:

  • Найменування - повне найменування томи для зберігання файлів;
  • Код - заповнюється автоматично;
  • Повний шлях
    • Для сервера 1С:Підприємства під управлінням Microsoft Windows - напишіть шлях до каталогу томи для сервера під керуванням Microsoft Windows у форматі UNC - \\Ім'я комп'ютера\Найменування папки\Найменування файлу\;

або

    • Для сервера 1С:Підприємства під управлінням Linux - напишіть шлях до каталогу для сервера під управлінням Linux;
  • Порядок заповнення - введіть ціле число, по умовчанню дорівнює 1. Заповнення томів здійснюється за порядком заповнення (за зростанням) до повного заповнення тома. Цю особливість можна використовувати, наприклад, для тимчасового відключення тома шляхом збільшення йому порядку заповнення.
  • Максимальний розмір - можна вказати максимальний розмір всіх файлів на томі (квота томи). Якщо сумарний розмір файлів на томі перевищує встановлену квоту, то буде використаний наступний по порядку. Якщо ні на одному з доступних томів не буде достатньо місця для розміщення файлу, то інформація про це буде записано в журнал реєстрації та буде видано повідомлення про помилку. Якщо квота встановлена рівною нулю, то при розміщенні файлів в томі вона не перевіряється; 
  • Поточний розмір, Мб - проставляється програмою автоматично по мірі наповнення тома, показує, наскільки том заповнений;
  • В полі Коментар напишіть детальну інформацію про том.

Особливості заповнення поля Повний шлях для Linux

  • Якщо сервер 1С:Підприємства працює під Linux, то шлях до того зберігання файлів можна вказати тільки локальний, наприклад, /home/path. Для того щоб використовувати UNC шлях, потрібно зробити на нього посилання. Для цього під правами адміністратора зробіть наступне:
    • виконайте команду виду mkdir folder, де замість "folder" напишіть ім'я свого каталогу. Наприклад, у каталозі /home виконайте команду mkdir shared - буде створено каталог shared з повним шляхом /home/shared.
    • виконайте команду виду mount -t cifs //server/share folder -o //domain/user, password=pwd, де
      - де //server/share - UNC шлях до іншого комп'ютера , folder - шлях до каталогу на комп'ютері Linux, наприклад \home\shared, //domain/user - ім'я домену/користувача, якщо необхідно, pwd - пароль користувача (якщо необхідно).
    •  Якщо ім'я і пароль не потрібні, команда виглядає так:  mount -t cifs //server/share folder.
  • Після цього у шляху до тому вказуємо folder, наприклад /home/shared.

Перегляд списку файлів в томі

  • У кожному томі файли зберігаються в підкаталогах. На кожну дату створення файлу створюється свій підкаталог. Всі файли зберігаються з оригінальним іменем та розширенням, між ними додається номер версії, наприклад "Наказ.2.txt".
  • Для того щоб переглянути список файлів у томах, виконайте команду Файли у томі на панелі навігації тома.
