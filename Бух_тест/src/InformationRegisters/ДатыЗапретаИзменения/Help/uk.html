Про заборону зміни (завантаження) даних

Іноді при роботі з інформаційною базою потрібно встановити заборону зміни будь-яких даних до певної дати. Наприклад, після здачі квартальної бухгалтерської звітності в податкову інспекцію, зазвичай встановлюється заборона зміни тих даних, які використовувалися для формування цієї звітності, тобто дані за останній день звітного періоду.
Заборона завантаження на відміну від заборони зміни може бути встановлений для даних, які повинні змінюватися тільки в одному вузлі інформаційної бази, а також за правилами, які випливають з вимог обміну даними, а не з типових бізнес-процесів організації.
Заборони завантаження даних можуть не використовуватися. У цьому випадку доступна тільки форма налаштування дати заборони зміни даних.

Прояви встановленої заборони

Якщо дата заборони зміни даних встановлена, то при спробі внести зміни в ІБ до дати заборони зміни користувачеві виводиться повідомлення про неможливість зміни даних. Якщо дату заборони зміни не встановлено, то обмеження не діють. Якщо у закритому періоді є об'єкти позначені на видалення, то їх вилучення буде неможливо.
Якщо дата заборони завантаження даних встановлена, то при спробі завантажити дані в ІБ до дати заборони завантаження всі заборонені для завантаження дані будуть пропущені, відомості про них будуть записані в журнал реєстрації, а дозволені дані будуть завантажені.

Можливості встановлення дат заборони

При встановленні дати заборони можна задавати не тільки довільну дату, але і відносну. Для відносних дат (крім значення Попередній день) на формі можна вказати кількість днів «відстрочки» до установлення дати заборони, протягом яких заборона все ще не буде діяти.
Відносні дати заборони повинні перераховуватися щодня. Для цього передбачено регламентне завдання. Однак для файлової ІБ і в інших випадках може знадобитися виконати перерахунок поточних значень відносних дат заборони вручну. Перерахунок можна виконати командою Перерахувати дати.

Якщо дату заборони необхідно встановити для всіх користувачів системи (для всіх вузлів планів обміну), то у полі Установлення дати заборони необхідно вибрати значення Для всіх користувачів (Для всіх вузлів планів обміну). При цьому у формі стануть доступні поля Спосіб зазначення дати заборони та Дата заборони.

Якщо різним користувачам (вузлів планів обміну) необхідно встановлювати різні значення дати заборони, то в полі Установлення дати заборони необхідно вибрати значення За користувачами (По вузлам планів обміну). При цьому у формі буде доступний список, в який можна додавати користувачів/групи користувачів (вузли планів обміну) і настроїти для кожного з них своє значення дати заборони. У списку по умовчанню завжди присутній елемент <Для всіх користувачів> (<Для всіх вузлів планів обміну>). Елемент не можна видалити або змінити. Настройки дати заборони виконані для цього елемента будуть використовуватися для всіх користувачів (вузлів планів обміну) крім тих, які містяться в списку. Для того що б виконати настройку дати заборони, необхідно виділити в списку потрібного користувача або групу користувачів (вузол плану обміну) або елемент <Для всіх користувачів> (<Для всіх вузлів планів обміну>), а самі настройки виконати нижче в полях Спосіб зазначення дати заборони і Дата заборони.

Спосіб зазначення дати заборони може приймати наступні значення:

  • Загальна дата – при цьому способі одна дата заборони задається для всіх елементів даних ІБ.
  • За розділами – для кожного розділу програми встановлюється своя дата заборони. Коли дата заборони задається за розділами, у формі доступний список, в який можна додавати Розділи (відповідною командою) та вказувати для кожного розділу своє значення дати заборони. У списку по умовчанню присутній елемент <Загальна дата>. Значення дати заборони для цього елемента буде використовуватися для всіх розділів по умовчанню крім тих, які вже містяться в списку. Якщо в полі Установлення дати заборони зазначено Для всіх користувачів (Для всіх вузлів планів обміну), вилучити елемент <Загальна дата> і розділи не можна. Якщо в полі Установлення дати заборони вказано За користувачами (По вузлам планів обміну), вилучити елемент <Загальна дата> і розділи можна.
  • По об'єктах – дати заборони задаються в розрізі об'єктів програми. Для кожного об'єкта встановлюється своя дата заборони. Якщо вибрано цей параметр, то в формі доступний список в який можна додавати об'єкти (відповідною командою) та вказувати для кожного об'єкта своє значення дати заборони. Так само як і в попередньому способі в списку для аналогічних цілей міститься елемент <Загальна дата>.
  • За розділами і об'єктами – при цьому способі дати заборони задаються і в розрізі розділів та в розрізі об'єктів. Коли дата заборони задається за розділами і об'єктами, у формі доступний список в який можна додавати розділи та об'єкти, вказуючи для кожного розділу і кожного об'єкту своє значення дати заборони. Так само як і в попередніх способах у списку міститься елемент <Загальна дата> для аналогічних цілей.

У звітах можлива групування даних за розділами/об'єктів: Звіт по розділах і об'єктів для користувачів (Звіт за розділами і об'єктів для вузлів планів обміну) або за користувачами (по вузлам планів обміну): Звіт по користувачам (Звіт по вузлам планів обміну). Звіт дозволяє побачити результуючі дати заборони, встановлені для користувачів/груп користувачів (вузлів планів обміну) з урахуванням пріоритетів.
