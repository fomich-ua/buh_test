Призначена для контролю і заміни правил конвертації даних. Правила конвертації даних використовуються при синхронізації даних двох програм. 

Настройка синхронізації даних по умовчанню проводиться у відповідності з так званими Типовими правилами обміну. Правила обміну складаються з правил конвертації об'єктів та правил реєстрації об'єктів.

Правила конвертації об'єктів можна відкрити з настройки синхронізації даних по команді Всі дії - Завантажити правила конвертації об'єктів.

Джерело правил

Для стандартної синхронізації даних між типовими конфігураціями використовуються Типові правила з конфігурації (ім'я макета правил конвертації, з якого були отримані типові правила, вказується). Однак у ряді випадків необхідно використовувати Правила з файлу (наприклад, якщо синхронізація ведеться між базами з нетиповими конфігураціями) або Режим налагодження (наприклад, якщо під час синхронізації даних сталася помилка).

За допомогою перемикача Джерело правил можна встановити, звідки брати правила реєстрації об'єктів:

  • Типові правила з конфігурації (обрані по умовчанню).
  • Правила з файлу. Використання правил, завантажених з файлу, може привести до помилок при переході на нову версію програми.
  • Режим налагодження. Якщо вибрано Режим налагодження, то стають доступними наступні настройки:
    • Підключити зовнішню обробку для налагодження вивантаження – вказується файл зовнішньої обробки, що містить текстовий модуль обробників вивантаження, сформований в конфігурації "Конвертація даних". У клієнт-серверному режимі роботи вказується шлях на сервері.
    • Підключити зовнішню обробку для налагодження завантаження – вказується файл зовнішньої обробки, що містить текстовий модуль обробників завантаження, сформований в конфігурації "Конвертація даних". У клієнт-серверному режимі роботи вказується шлях на сервері.
    • Режим протоколювання синхронізації даних – якщо включено, то в процесі обміну даними події та помилки, що виникають, будуть записуватися у вказаний файл. Якщо файл не вказано, то повідомлення будуть виводитися в Журнал реєстрації.
    • Не зупиняти виконання обміну даними при виникненні помилки – якщо встановлено, то виконання синхронізації даних не буде зупинено при виникненні будь-якої помилки. Обмін даними завершиться до кінця з виведенням налагоджувальних повідомлень у файл протоколу обміну або журнал реєстрації.

Нижче виводиться найменування правил, які в даний момент використовуються.

Інформація про правила

Для контролю виводиться інформація про правила:

  • Попередження про можливість виникнення помилок при використанні правил з зовнішнього файлу;
  • Правила конвертації цієї програми від - вказується дата і час створення правил;
  • Конфігурація джерело - вказується версія правил конвертації об'єктів;
  • Конфігурація приймач - вказується версія правил конвертації об'єктів;
  • Правила конвертації кореспондента від - вказується дата і час створення правил;
  • Конфігурація джерело - вказується версія правил конвертації об'єктів;
  • Конфігурація приймач - вказується версія правил конвертації об'єктів.

Додатково

  • Увімкніть прапорець Використовувати фільтр вибіркової реєстрації об'єктів.

Завантаження правил

  • Встановіть перемикач Джерело правил в потрібне положення.
  • Натисніть Завантажити правила.
  • За необхідності вкажіть шлях до файлу з правилами.

Вивантаження правил

За необхідності правила реєстрації можна вивантажити в файл.

  • Для цього натисніть Вивантажити правила.
  • Виберіть спосіб вивантаження:
    • Звичайним файлом;
    • В архіві .zip.
  • Виберіть шлях до файлу на комп'ютері або на мережевому диску, найменування проставляється програмою по умовчанню. Натисніть Зберегти.

Заміна типових правил конвертації об'єктів правилами з зовнішнього файлу

  • Встановіть значення перемикача Джерело правил в положення Правила з файлу або Режим налагодження і по кнопці Завантажити правила виберіть файл правил на комп'ютері або в мережній папці.
  • Зверніть увагу, що завантажені коректні правила, відповідні для використовуваних версій програми-джерела і програми-приймача. Для цього виводиться Інформація про правила, де вказано, за яку дату ці правила. 
  • Натисніть на кнопку Записати і закрити. Завантажені правила будуть застосовані негайно.

Примітка: правила конвертації об'єктів і правила реєстрації об'єктів використовуються для всіх однотипних синхронізацій даних (які ґрунтуються на одному плані обміну; наприклад: на базі плану обміну Повний налаштовані дві синхронізації даних "Синхронізація з філією в Краснодарі" і "Синхронізація з представництвом у Києві"). Тому якщо в програмі встановлено кілька однотипних синхронізацій, то заміну правил достатньо виконати одноразово (вибравши будь-яку з налаштованих синхронізацій даних).
