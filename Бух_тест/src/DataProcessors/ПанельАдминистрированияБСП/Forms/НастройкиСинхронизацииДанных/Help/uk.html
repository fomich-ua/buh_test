У даній панелі здійснюється настройка синхронізації даних з іншими програмами та організація спільної роботи в розподіленій інформаційній базі.

  • Синхронізація даних
    		•  Прапорець синхронізації даних з іншими програмами, наприклад з 1С:Управління торгівлею для України, редакція 3.0.
  • Синхронізація даних
    		•  Настройка та виконання синхронізації даних з іншими програмами.
  • Попередження при синхронізації даних
    		•  Перегляд непроведених документів, незаповнених реквізитів, конфліктів і пропущених по даті заборони.
  • Префікс цієї інформаційної бази
    		•  Префікс додається автоматично до кодів елементів довідників та номерами документів для того, щоб розрізняти програми, в яких вони були введені, і уникнути дублювання даних.
  • Дати заборони завантаження даних
    		•  Встановлення заборони завантаження даних минулих періодів з інших програм. Див. також: Дати заборони зміни даних в Підтримка та обслуговування.

  • Каталог тимчасових файлів Windows
    		•  Шлях до єдиного каталогу тимчасових файлів всіх серверів кластера, який буде використовуватися для серверів, що працюють під управлінням ОС Windows.
  • Каталог тимчасових файлів Linux
    		•  Шлях до єдиного каталогу тимчасових файлів всіх серверів кластера, який буде використовуватися для серверів, що працюють під управлінням ОС Linux.

    Каталоги тимчасових файлів можна не заповнювати, коли в кластері 1С:Підприємства працює тільки один сервер або програма працює у файловому режимі.
    Якщо всі сервери кластера працюють під управлінням операційних систем сімейства Windows (Windows/Linux), то друге поле заповнювати не обов'язково.

    Каталог повинен бути доступний на читання і запис для всіх серверів кластера. В тому числі - для всіх користувачів операційних систем, від імені яких запускаються сервера 1С:Підприємства. Так само UNC шлях до нього повинен бути однаковим для всіх серверів одного сімейства - для цього каталог необхідно підключати до серверів під одним ім'ям.

    У полі Каталог тимчасових файлів Linux можна вказати тільки локальний шлях, наприклад, /home/path.

    Щоб використовувати UNC шлях, потрібно зробити на нього посилання. Для цього під правами адміністратора (root) зробіть наступне:

    mkdir folder
    --- замість folder напишіть ім'я каталогу. наприклад в каталозі /home робимо mkdir shared - створить каталог shared з повним шляхом /home/shared.

    mount -t cifs //server/share folder -o //domain/user, password=pwd
    - де //server/share - UNC шлях до іншого комп'ютера , folder - шлях каталогу на комп'ютері Linux, наприклад \home\shared //domain/user - ім'я домену/користувача, якщо потрібно, pwd - пароль користувача (якщо потрібно).

    Якщо ім'я і пароль не потрібні, команда виглядає так:
    mount -t cifs //server/share folder

    Після цього в дорозі до того вказуємо folder, наприклад, /home/shared
