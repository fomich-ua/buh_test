Форма настройки системи резервного копіювання інформаційної бази.

Пропонує наступні можливості:

1) Виконувати резервне копіювання автоматично.(Доступний тільки для файлового режиму інформаційної бази). При виборі даного пункту, програма запропонує вибрати розклад резервного копіювання декількох найбільш часто використовуваних варіантів, а також можливість вибору свого власного варіанту розкладу ( див. Розклад регламентного завдання). Також необхідно вибрати каталог, куди будуть поміщатися резервні копії, і, при необхідності параметри очищення даного каталогу від застарілих резервних копій. При завершенні настройки система перейде в режим автоматичного резервного копіювання.

2) Копіювання виконується сторонніми засобами. Дозволяє повністю відмовитися від використання даного механізму. Рекомендується вибирати тільки в разі використання клієнт-серверного режиму інформаційної бази, або при використанні сторонніх програм копіювання.

3) Нагадати про резервне копіювання пізніше. Даний пункт обирається в тому випадку, якщо немає можливості чітко визначити, наскільки інші пункти меню актуальні. У даному випадку система запропонує ввести проміжок часу, по закінченню якого знову запропонує вам налаштувати резервне копіювання. Без вибору цього пункту попередження про необхідність налаштувати резервне копіювання будуть з'являтися кожну годину.

Якщо резервне копіювання завершилося невдачею, а в часовому log-файлі є запис: Виняток у createConnection: Error, Automation server can't create object то, швидше за все, використовується 32-розрядна компонента comcntr.dll на комп'ютері під керуванням 64-розрядної операційної системи. Для усунення даної проблеми необхідно взяти 64-розрядну компоненту comcntr.dll з постачання 1С Підприємство x64 Сервер і зареєструвати її, запустивши з командного рядка:

regsvr32 <каталог 64-розрядного comcntr.dll>\comcntr.dll

Важливо: для того, щоб увійти в інформаційну базу, якщо резервне копіювання не здійснилось, необхідно запустити 1С:Підприємство з параметрами: ENTERPRISE /F <шлях до інформаційної бази> /СРазрешитьРаботуПользователей /UC РезервноеКопирование.
